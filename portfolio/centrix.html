Objective

Identify pain points in the presentation of Centrix Solutionsâ€™ company website.

Target User

Prospective customer from a community bank who is searching for information about Centrix products.

Key Function

In order to identify the key function of the website, I prioritized tasks based on how important I expected them to be to users and to Centrix's bottom line. I used this prioritization scheme to focus user testing on the highest-impact task.

Tasks

After identifying the need to find product information as the key function of the website, I started users at centrixsolutions.com and asked them to find a product that helps prevent fraud.

Common Findings

"Are Centrix products still available?"
"Do I need to go to the Q2 or Centrix website for more information?"

The first page of the website is dedicated to Centrix being a Q2 company. The primary action on this page is to "Learn More" about the acquisition, which takes the user to a press release on the Q2 website, where there is no information about Centrix products. This 'Learn More' button can be hidden behind the secondary action if the viewport height is short enough, which completely breaks the purpose of this page (to provide more information about the acquisition).

"This is annoying. It seems like they're more proud of their owners than their products."

Once on the Centrix website and browsing, the user expects to be able to click the company logo to return to the home page. Instead, they are taken back to the splash page, which increases the user's frustration each time they land on it.

Recommendation

Skip the acquisition page to bring focus back to the task with the most value - providing product information. An alternative to informing people of the acquisition could be to provide a link to the press release on the home page or in the news section and to make the branding more consistent between Centrix and Q2.

Solution

To increase consistency between the Q2 and Centrix websites, header and footer styles should match. Since we will be using the Q2 website styles to guide us, there is no need to create wireframes or prototypes. Instead, I went straight to a high-fidelity HTML/CSS prototype.

Next Steps *

There are a number of changes needed to further encourage consistency between the two websites. Below is a list of changes on the home page that have been identified so far - some related to style consistency, others related to finding product information. Aside from this list, user feedback and website analytics will help determine new tasks as well as their priorities.

  • Clarify product value on home page
  • Difficult to read 'a Q2 Company' in Centrix logo
  • Update fonts to match Q2
  • Update sub-menu styles and animations to match Q2
  • Update content styles to match Q2
  • Update slideshow styles and animations to match Q2
  • Update responsive menu styles to match Q2
  • Update responsive footer styles to match Q2
  • Have to wait for slide for product information
  • Accessibility issues: missing alt tags for images
  • Slideshow has most product information but user has to wait for slide to see details
  • Icons for all products are the same, which causes confusion with company logo
  • Add slide for acquisition information
  • Add acquisition press release to News Page
  • Fix page width issues on smaller screens
